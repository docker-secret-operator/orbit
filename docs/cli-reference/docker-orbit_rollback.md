## docker-orbit rollback

Restore traffic to the previous version after a failed deployment

### Synopsis

Reads the last rollout state for the service, re-registers the old
backend with the proxy, drains the new (failing) backend, and removes it.

The rollout/deploy commands save a state file to
/tmp/orbit-<service>-state.json between the point the new backend is
registered and the old one is removed. If the new deployment fails, run this
command immediately to restore traffic.

Example:
  docker-orbit rollback web
  docker-orbit rollback web --dry-run
  docker-orbit rollback web --yes --json
  docker-orbit rollback web --drain 15s --control-addr http://localhost:9901

```
docker-orbit rollback <service> [flags]
```

### Options

```
      --api-token string      Control API bearer token
      --control-addr string   Override proxy control API address from state
  -d, --drain duration        Override drain period from state
      --dry-run               Show the rollback plan without executing it
  -h, --help                  help for rollback
      --json                  Output as JSON
      --to string             Roll back to a specific generation/backend ID (see limitations in --help)
  -y, --yes                   Skip the confirmation prompt (required for non-interactive/CI use)
```
