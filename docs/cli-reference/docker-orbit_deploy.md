## docker-orbit deploy

Safely deploy a new version of a service (plan, confirm, execute)

### Synopsis

Deploys a new version of the named service using the same zero-downtime
engine as 'docker orbit rollout', with production safety built in:

  1. Pre-flight checks (Docker reachable, compose file valid, proxy healthy,
     recovery state consistent) — the same checks 'docker orbit doctor' runs.
  2. A deployment plan preview (current generation, backend health, what will
     happen) shown before anything changes.
  3. A confirmation prompt, unless --yes or --dry-run is given.
  4. Live progress reporting through each phase as it happens.
  5. A completion summary with the resulting generation and health state.

Use --dry-run to see the plan without deploying anything.

Example:
  docker orbit deploy web
  docker orbit deploy web --dry-run
  docker orbit deploy web --yes --json          # non-interactive, for CI
  docker orbit deploy web --project myapp       # verify the target proxy first

```
docker-orbit deploy <service> [flags]
```

  -d, --drain duration        Drain period before removing old container (default 5s)
      --dry-run               Show the deployment plan without executing it
  -f, --file string           docker-rollout compose file (default "docker-rollout-compose.yml")

      --project string        Verify the queried proxy reports this service/project name
      --pull                  Pull latest image before deploying
      --stability duration    How long to watch the new backend before draining the old one; auto-rolls back if it fails (default 10s)
  -t, --timeout duration      Healthcheck timeout (default 1m0s)
  -y, --yes                   Skip the confirmation prompt (required for non-interactive/CI use)
