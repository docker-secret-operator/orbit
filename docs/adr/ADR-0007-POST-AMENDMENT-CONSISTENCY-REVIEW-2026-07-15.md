# ADR-0007 Post-Amendment Consistency Audit

**Date:** 2026-07-15
**Scope:** Part 1 of the post-amendment review — find and fix stale text left over from ADR-0007's original draft after the acceptance-review amendment (§10's withdrawal of the `ORBIT_PROXY_INSTANCE` redefinition). Documentation-only. No Go code touched. The accepted architecture (Compose project as isolation boundary, self-inspection for proxy identity, ownership tuple, network identity as operational-only, fail-closed startup) is not reopened here — every item below is either a stale reference to the withdrawn decision or a plain cross-reference numbering error, not a challenge to the accepted decisions.

## Method

Read `docs/adr/ADR-0007-backend-identity-and-ownership.md` end to end after the amendment, then grep-verified every `§N` cross-reference in the file against the actual current section it points to, to catch numbering drift the amendment's targeted edits could have left behind (the amendment edited §7, §9, §10, §12, §15, §16, §22, §23, §27, and the Final Decision Answers — every other section's *cross-references into those* needed independent verification, since editing a section's content doesn't relabel who points at it).

All 16 are documentation-only corrections. None of them touch or reopen the accepted architecture: Compose project as the isolation boundary, `com.docker.compose.project` as authoritative, self-inspection as the proxy's identity source, `ORBIT_PROXY_INSTANCE` unchanged, `orbit.io/proxy-instance` removed, the `<project>-<service>-<container-id>` backend ID shape, the four-part ownership tuple, network identity as operational-only, and fail-closed startup semantics are all reaffirmed exactly as accepted, not modified, by every correction above.

## Disposition

All 16 corrections applied directly to `docs/adr/ADR-0007-backend-identity-and-ownership.md`. No architecture change. No code change.
